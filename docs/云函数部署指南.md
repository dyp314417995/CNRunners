# 云函数部署指南

## 前提条件

1. 已开通微信云开发
2. 已安装微信开发者工具
3. 项目已导入微信开发者工具

---

## 一、部署步骤

### 1.1 打开云开发控制台

在微信开发者工具中，点击顶部「云开发」按钮，进入云开发控制台。

### 1.2 创建数据库集合

在云开发控制台中，进入「数据库」，创建以下集合：

| 集合名称 | 说明 |
|----------|------|
| `users` | 用户表 |
| `groups` | 群组表 |

**注意**：users集合需要设置权限为「自定义安全规则」，添加以下规则：

```json
{
  "read": true,
  "write": "doc._id == auth.openid || auth.openid in get(#database/users).data[_id]._id"
}
```

或者暂时设置为「所有人可读，仅创建者可写」进行测试。

### 1.3 创建群组初始数据

在 `groups` 集合中添加以下数据：

```json
[
  {
    "_id": "group_1",
    "name": "一群",
    "isEnabled": true,
    "sort": 1
  },
  {
    "_id": "group_2",
    "name": "二群",
    "isEnabled": true,
    "sort": 2
  },
  {
    "_id": "group_3",
    "name": "分舵群",
    "isEnabled": false,
    "sort": 3
  }
]
```

### 1.4 部署云函数

在微信开发者工具中，找到 `cloudfunctions` 目录，对以下每个云函数文件夹右键选择「上传并部署」：

#### 需要部署的云函数列表

| 云函数路径 | 功能 |
|------------|------|
| `cloudfunctions/user/login` | 用户登录（已存在） |
| `cloudfunctions/user/register` | 用户注册 |
| `cloudfunctions/group/getList` | 获取群组列表 |
| `cloudfunctions/admin/getMembers` | 获取成员列表 |
| `cloudfunctions/admin/updateMember` | 更新成员信息 |
| `cloudfunctions/admin/getAdmins` | 获取管理员列表 |
| `cloudfunctions/admin/updateAdmin` | 添加/撤销管理员 |
| `cloudfunctions/admin/updateGroup` | 更新群组设置 |

**部署步骤**：
1. 右键点击云函数文件夹
2. 选择「上传并部署：云端安装依赖」
3. 等待部署完成

## 二、云函数权限说明

### 2.1 超管身份验证

由于微信云函数默认可以获取到调用者的 `openid`，我们通过 `openid` 来识别用户身份：

- `super_admin`：第一个注册的用户，系统自动设置为超管
- `admin`：通过超管提升的管理员
- `member`：普通成员

### 2.2 权限检查逻辑

每个管理云函数内部都会检查调用者身份：

```javascript
// 获取当前调用者的openid
const wxContext = cloud.getWXContext()
const openid = wxContext.OPENID

// 查询当前用户信息
const currentUser = await usersCollection.doc(openid).get()

// 检查角色
if (currentUser.data.role !== 'admin' && currentUser.data.role !== 'super_admin') {
  return { success: false, error: '权限不足' }
}
```

## 三、常见问题

### Q1: 部署云函数时报错 "找不到模块"

**原因**：依赖包未安装

**解决**：确保选择「上传并部署：云端安装依赖」选项

### Q2: 云函数调用失败

**排查步骤**：
1. 检查云开发控制台「云函数」页面，确认函数已上传
2. 查看「日志」 tab 获取详细错误信息
3. 检查数据库集合是否已创建

### Q3: 数据库权限不足

**解决**：在云开发控制台调整集合权限为「仅创建者可写」或添加自定义安全规则

### Q4: 用户角色未正确识别

**原因**：初始用户（第一个注册的用户）需要手动设置为 `super_admin`

**解决**：在 `users` 集合中，找到第一个用户的记录，将其 `role` 字段设置为 `super_admin`

## 四、功能测试清单

部署完成后，请按以下清单测试功能：

### 4.1 用户注册
- [ ] 打开小程序，进入登录页
- [ ] 点击「微信一键登录」
- [ ] 授权获取昵称头像
- [ ] 填写手机号、姓名，选择群组
- [ ] 点击「完成注册」
- [ ] 验证 users 集合中是否有新记录
- [ ] 验证该用户的 role 是否为 super_admin

### 4.2 管理员管理
- [ ] 以超管身份进入「我的」页面
- [ ] 点击「管理员管理」
- [ ] 验证管理员列表是否正确
- [ ] 测试「群组设置」Tab
- [ ] 启用/禁用分舵群
- [ ] 返回注册页，新用户是否能看到分舵群选项

### 4.3 成员管理
- [ ] 进入「成员管理」页面
- [ ] 测试搜索功能
- [ ] 测试群组筛选
- [ ] 点击成员查看详情
- [ ] 提升普通成员为管理员

## 五、云函数更新

如果后续修改了云函数代码，需要重新部署：

1. 修改云函数代码
2. 右键云函数文件夹
3. 选择「上传并部署：云端安装依赖」
4. 等待部署完成（通常几秒）
